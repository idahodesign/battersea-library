Getting Started

Quick Installation

Include the CSS and JavaScript files in your project:

<!-- CSS -->
<link rel="stylesheet" href="path/to/battersea-library-min.css">

<!-- Core JavaScript (Required) -->
<script src="path/to/battersea-utils.js"></script>
<script src="path/to/battersea-core.js"></script>

<!-- Component Scripts (Load only what you need) -->
<script src="path/to/battersea-slider.js"></script>
<script src="path/to/battersea-accordion.js"></script>

Or clone from GitHub:

git clone https://github.com/idahodesign/battersea-library.git

Simple Usage Example

Components are initialized automatically using data attributes:

<!-- Accordion Example -->
<div data-accordion>
  <div class="battersea-accordion__item">
    <button data-accordion-trigger>Section 1</button>
    <div data-accordion-content>
      <p>Content goes here...</p>
    </div>
  </div>
</div>

<!-- Slider Example -->
<div data-slider data-slider-autoplay="true">
  <div class="battersea-slider__track">
    <div class="battersea-slider__slide">Slide 1</div>
    <div class="battersea-slider__slide">Slide 2</div>
  </div>
</div>

No JavaScript configuration required! Components auto-initialize on page load.

Theme Customization

Customize the library using CSS variables or LESS variables:

/* CSS Custom Properties */
:root {
  --battersea-primary: #667eea;
  --battersea-secondary: #764ba2;
  --battersea-text: #333;
}

/* Or use LESS variables */
@primary-color: #667eea;
@secondary-color: #764ba2;

/* Dynamic theming with less.js */
less.modifyVars({
  '@primary-color': '#ff6b6b',
  '@secondary-color': '#4ecdc4'
});

Frequently Asked Questions

Everything you need to know about Battersea Library

Do I need to include all components?

No! That's the beauty of Battersea's modular architecture. You only need to include battersea-utils.js and battersea-core.js, then add only the component scripts you actually use. This keeps your page size minimal and performance optimal.

For example, if you only use Tabs and Accordion, just include those two component files along with the core files.

Is Battersea Library free to use?

Yes! Battersea Library is completely free and open source under the MIT License. You can use it in personal projects, commercial projects, modify it, redistribute it - whatever you need. No attribution required (though always appreciated!).

Can I use this with React, Vue, or other frameworks?

Absolutely! While Battersea is built with vanilla JavaScript, it works alongside any framework. The components use standard DOM manipulation and can be integrated into React components, Vue templates, or any other framework.

Just initialize the components after your framework renders the DOM elements. Many developers use Battersea components for specific UI elements while keeping their main app in a framework.

How do I customize the styling?

You have multiple options:

  • CSS Custom Properties: Override variables like --battersea-primary in your own CSS
  • LESS Variables: Modify the LESS source files and recompile
  • Runtime Theming: Use less.js to change themes dynamically (like the customizer on this page!)
  • Custom CSS: Add your own classes and override specific styles

Are the components accessible?

Yes! Accessibility is a core principle of Battersea Library. Every component includes:

  • Proper ARIA labels and roles
  • Full keyboard navigation support
  • Screen reader compatibility
  • Focus management and visible focus indicators
  • Semantic HTML structure

We follow WCAG 2.1 Level AA guidelines as a minimum standard.

What browsers are supported?

Battersea supports all modern browsers:

  • Chrome (latest + last 2 versions)
  • Firefox (latest + last 2 versions)
  • Safari (latest + last 2 versions)
  • Edge (latest + last 2 versions)
  • Mobile browsers (iOS Safari, Chrome Mobile, Samsung Internet)

We use progressive enhancement, so core functionality works even in older browsers, with enhanced features in modern ones.

Can I contribute to the library?

We'd love your contributions! Check out the GitHub repository to:

  • Report bugs or request features via Issues
  • Submit pull requests with improvements
  • Improve documentation
  • Share examples of your projects using Battersea

How do I get support?

For support and questions:

  • Check the Getting Started guide above
  • Browse the individual component demo pages for detailed examples
  • Open an issue on GitHub
  • Review the source code - it's well-commented and easy to follow
