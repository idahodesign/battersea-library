Key Features

⇵

Row Reordering

Grab the drag handle and move rows up or down. Works with both mouse and touch input.

⇆

Column Reordering

Drag column headers left or right to rearrange them. Disabled on mobile where tables stack into cards.

⚲

Text Filter

Type to search across every cell in the table. Results update as you type with debounced input.

▼

Column Filters

Per-column dropdown filters showing unique values. Select a value to show only matching rows.

💾

Persistence

Order is saved to sessionStorage and restored on page load. Events fire for database saving too.

⌨

Keyboard Support

Focus a row grip and use arrow keys to move it up or down. Screen reader announcements included.

Configuration & Options

Data Attributes

Configure the DragTable component using these data attributes on the container element:

Attribute Values Default Description
data-drag-table - required Initialises the DragTable component
data-dragtable-reorder rows / columns / both rows Which axis can be reordered by dragging
data-dragtable-filterable true / false false Show the global text search input
data-dragtable-column-filters true / false false Show per-column dropdown filters
data-dragtable-striped true / false true Alternating row background colours
data-dragtable-responsive true / false true Stacked card layout on mobile, scroll hints on wider screens
data-dragtable-storage-key String - SessionStorage key for persisting the order. Omit to disable persistence

Custom Events

The component fires these events on the container element:

Event Detail Description
battersea:dragTableReorder { columnOrder, rowOrder, columns, rows } Fired when rows or columns are reordered. Contains the full order data for saving to a database
battersea:dragTableFilter { query, results } Fired when the text filter changes

Public Methods

Access the component instance via Battersea.getInstance(element):

Method Returns Description
getOrderData() Object Returns the current column and row order as a plain object, ready for sending to an API
destroy() - Removes all event listeners and DOM elements

Usage Examples

Basic Row Reordering

Write a standard HTML table and the component adds drag handles automatically:

<div data-drag-table data-dragtable-reorder="rows">
  <table>
    <thead>
      <tr>
        <th>Name</th>
        <th>Role</th>
      </tr>
    </thead>
    <tbody>
      <tr><td>Alice</td><td>Developer</td></tr>
      <tr><td>Bob</td><td>Designer</td></tr>
    </tbody>
  </table>
</div>
Column Reordering

Set data-dragtable-reorder="columns" and drag the headers to rearrange:

<div data-drag-table data-dragtable-reorder="columns">
  <table>
    <thead>
      <tr>
        <th>Name</th>
        <th>Department</th>
        <th>Location</th>
      </tr>
    </thead>
    <tbody>
      <tr><td>Alice</td><td>Engineering</td><td>London</td></tr>
    </tbody>
  </table>
</div>
Full-Featured with Filters

Enable everything: row and column dragging, text search, and per-column filters:

<div data-drag-table
  data-dragtable-reorder="both"
  data-dragtable-filterable="true"
  data-dragtable-column-filters="true"
  data-dragtable-striped="true"
  data-dragtable-storage-key="my-table">
  <table>
    <thead>...</thead>
    <tbody>...</tbody>
  </table>
</div>
Saving to a Database

Listen for the reorder event and post the data to your API:

var table = document.querySelector('[data-drag-table]');

table.addEventListener('battersea:dragTableReorder', function(e) {
  // e.detail contains the full order data
  console.log('Row order:', e.detail.rowOrder);
  console.log('Column order:', e.detail.columnOrder);

  // Send to your API
  fetch('/api/save-order', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(e.detail)
  });
});

// Or call getOrderData() at any time
var instance = Battersea.getInstance(table);
var orderData = instance.getOrderData();
