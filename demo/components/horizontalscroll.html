Key Features

▭

Content-Agnostic

Put any HTML inside panels. The component handles the scroll mechanics, you handle the content.

•••

Progress Dots

Clickable dots show which panel is active. Enable or disable them per zone with a data attribute.

⇄

Direction Control

Scroll right-to-left (default) or left-to-right. Set the direction per zone to suit your layout.

☍

Responsive

On mobile, panels stack vertically with normal page scroll. No horizontal mechanics below 768px.

⚡

Performance

Throttled scroll handlers and GPU-composited transforms keep everything smooth at 60fps.

×

Clean Destroy

Full cleanup on removal. Event listeners, injected DOM, and inline styles are all tidied up.

Configuration & Options

Data Attributes

Configure the component using these data attributes on the container element:

Attribute Values Default Description
data-horizontal-scroll - required Initialises the horizontal scroll component
data-horizontalscroll-dots true / false true Show or hide progress indicator dots
data-horizontalscroll-dot-position number (px) 20 Distance from the bottom of the viewport
data-horizontalscroll-speed 0.5 - 2 1 Scroll speed multiplier. Higher values scroll through panels faster.
data-horizontalscroll-direction right / left right Scroll direction. "right" slides panels left (default). "left" reverses the direction.

HTML Structure

Each panel is a direct child with the data-horizontal-scroll-panel attribute:

<div data-horizontal-scroll>
  <div data-horizontal-scroll-panel>
    <!-- Any content: images, text, cards, forms -->
  </div>
  <div data-horizontal-scroll-panel>
    <!-- Panel content -->
  </div>
</div>

CSS Classes

The component manages these classes automatically:

Class Element Description
battersea-hscroll Root element Applied to the container on initialisation
battersea-hscroll--mobile Root element Applied below 768px to stack panels vertically
battersea-hscroll__panel Each panel Applied to panel children for sizing
battersea-hscroll__dot--active Dot button Marks the currently visible panel's dot

Usage Examples

Basic Usage

The simplest setup with default options:

<div data-horizontal-scroll>
  <div data-horizontal-scroll-panel>
    <h2>Panel One</h2>
    <p>Your content here</p>
  </div>
  <div data-horizontal-scroll-panel>
    <h2>Panel Two</h2>
    <p>More content</p>
  </div>
  <div data-horizontal-scroll-panel>
    <h2>Panel Three</h2>
    <p>Even more content</p>
  </div>
</div>
Reversed Direction

Scroll left-to-right instead of the default right-to-left:

<div data-horizontal-scroll
     data-horizontalscroll-direction="left">
  <div data-horizontal-scroll-panel>Panel 1</div>
  <div data-horizontal-scroll-panel>Panel 2</div>
  <div data-horizontal-scroll-panel>Panel 3</div>
</div>
Custom Speed and Dot Position

Adjust how fast panels scroll and where the dots sit:

<div data-horizontal-scroll
     data-horizontalscroll-speed="1.5"
     data-horizontalscroll-dot-position="50">
  <div data-horizontal-scroll-panel>Fast panel 1</div>
  <div data-horizontal-scroll-panel>Fast panel 2</div>
</div>
Dots Disabled

Hide the progress dots entirely:

<div data-horizontal-scroll
     data-horizontalscroll-dots="false">
  <div data-horizontal-scroll-panel>Panel 1</div>
  <div data-horizontal-scroll-panel>Panel 2</div>
</div>
Listening to Events

Listen for panel changes to integrate with other functionality:

const zone = document.querySelector('[data-horizontal-scroll]');

zone.addEventListener('battersea:horizontalScrollChange', (e) => {
  console.log('Panel:', e.detail.panelIndex);
  console.log('Total:', e.detail.panelCount);
});
