Video Player Component

A fully custom video player with overlay controls, fullscreen support, seek bar, volume slider and auto-hiding controls bar — no browser defaults, everything stylable

Live Demonstrations

See the video player in action with different configurations

Basic Player

The simplest setup — just point it at a video file and the player builds itself. Click the video or the play button to start. Controls auto-hide after 3 seconds during playback.

Player with Poster and Title

Add a poster image that displays before playback, a title overlay in the top-left corner, and a custom accent colour via data-video-color.

Looping Muted Player

A looping, muted video with always-visible controls. Useful for background video or product demos where you want the controls to stay on screen.

Key Features

▶

Custom Controls

Every control is a real HTML element you can style with CSS. The native video controls are hidden — you're in full control of the look.

↔

Seek & Scrub

Click anywhere on the progress bar to jump to that point, or drag the handle to scrub through the video in real time.

♫

Volume Control

Adjustable volume slider with a mute toggle. Three volume icons reflect the current level — muted, low, and full.

⛶

Fullscreen

Enter and exit fullscreen with a single click or press F. Uses the Fullscreen API with vendor prefixes for cross-browser support.

A11y

Accessible

ARIA roles on all controls, full keyboard navigation (Space, arrows, F, M, Escape), and visible focus indicators throughout.

{}

Data Attributes

Configure source, poster, title, volume, looping, controls behaviour and accent colour all from the HTML. No JavaScript setup needed.

Configuration & Options

Data Attributes

Configure the video player using these data attributes on the container element:

Attribute Values Default Description
data-video - required Initialises the video player on the container element
data-video-src URL string required Path to video file (MP4, WebM, OGG)
data-video-poster URL string - Poster image displayed before playback starts
data-video-title string - Title overlay displayed in the top-left corner
data-video-autoplay true / false false Auto-start playback (may be blocked by browser unless muted)
data-video-loop true / false false Loop the video continuously
data-video-muted true / false false Start with video muted
data-video-volume 0–100 80 Initial volume level
data-video-preload none / metadata / auto metadata How much video data to preload
data-video-color CSS colour #667eea Accent colour for progress fill and active states
data-video-controls auto / visible auto Controls auto-hide after 3s (auto) or stay visible (visible)
data-video-fullscreen true / false true Show or hide the fullscreen button

Basic Markup

The minimum markup to get a video player running:

<div data-video data-video-src="path/to/clip.mp4"></div>

Custom Events

Listen for these events to hook into playback:

Event Detail Description
battersea:videoPlay { currentTime } Fired when playback starts
battersea:videoPause { currentTime } Fired when playback pauses
battersea:videoSeek { currentTime, percentage } Fired when the user seeks to a new position
battersea:videoVolumeChange { volume } Fired when volume changes
battersea:videoMute { muted } Fired when mute is toggled
battersea:videoEnded {} Fired when the video finishes playing
battersea:videoFullscreen { fullscreen } Fired when entering or exiting fullscreen

Usage Examples

Basic Video Player

The simplest possible setup — one element, one attribute:

<div data-video
  data-video-src="path/to/clip.mp4">
</div>
Player with Poster and Custom Colour

Show a poster image before playback, with a title overlay and teal accent:

<div data-video
  data-video-src="path/to/clip.mp4"
  data-video-poster="poster.jpg"
  data-video-title="My Video"
  data-video-color="#4ecdc4">
</div>
Looping Background Video

A muted, looping video with always-visible controls:

<div data-video
  data-video-src="background.mp4"
  data-video-loop="true"
  data-video-muted="true"
  data-video-controls="visible">
</div>
Listening to Video Events

Hook into playback events for custom behaviour:

const player = document.querySelector('[data-video]');

player.addEventListener('battersea:videoPlay', (e) => {
  console.log('Playing at:', e.detail.currentTime);
});

player.addEventListener('battersea:videoEnded', () => {
  console.log('Video finished');
});

player.addEventListener('battersea:videoFullscreen', (e) => {
  console.log('Fullscreen:', e.detail.fullscreen);
});
