Priority Fixes

Components where keyboard users are blocked or screen readers get no useful information.

Critical

  • Accordion -- Headers are divs with no tabindex, role, or keydown handler. Completely inaccessible to keyboard users.
  • Tabs -- No ARIA tabs pattern. No keyboard navigation between tabs. Needs role="tablist", role="tab", role="tabpanel".
  • Popup -- No role="dialog", no focus trap, no focus management. Modal is invisible to screen readers.
  • Tooltip -- No aria-describedby or role="tooltip". Screen readers cannot access tooltip content despite focus-triggered display.
  • Flipbox -- No keyboard access, no ARIA, no focus. Completely inaccessible.

High

  • ProgressBar / NestedProgress -- Missing role="progressbar" and all associated ARIA value attributes.
  • Graph -- No keyboard navigation, no data accessible to screen readers beyond chart title. Needs a hidden data table alternative.
  • DragDrop -- Base component has no keyboard support. Only the MiniQuiz integration adds it.
  • ProfileGrid -- Lightbox lacks dialog semantics, focus trap, and focus management.

Medium

  • Header / HorizontalNav -- Desktop dropdowns lack arrow key navigation and ARIA menu roles.
  • Slider / MultiSlider -- Missing carousel ARIA pattern (aria-roledescription, aria-live for slide changes).
  • ImageGallery -- Lightbox lacks focus trap (otherwise excellent).
  • Timeline -- Container needs tabindex="0" for keyboard events to work.
  • VideoBg -- Decorative video needs aria-hidden="true".
