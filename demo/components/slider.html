Configuration & Options

Data Attributes

Configure the slider using these data attributes on the container element:

Attribute Values Default Description
data-slider - required Initialises the slider component on the container element
data-slider-transition slide / fade slide The animation style between slides
data-slider-autoplay true / false false Whether slides advance automatically
data-slider-interval milliseconds 5000 Time between auto-advances (only used when autoplay is on)
data-slider-arrows true / false true Show or hide the previous/next arrow buttons
data-slider-dots true / false true Show or hide the dot indicators at the bottom

Element Structure

Each slider needs a track, slide items, and optionally navigation buttons and a dots container:

Attribute Element Description
data-slider-track div The track that holds all slide items
data-slider-item div Each individual slide (direct children of the track)
data-slider-prev button The "previous slide" navigation button
data-slider-next button The "next slide" navigation button
data-slider-dots div Container where dot indicators are generated

Basic Markup

Here's the minimum markup you need to get a slider running:

<div data-slider data-slider-transition="slide">
  <div data-slider-track>
    <div data-slider-item>Slide 1 content</div>
    <div data-slider-item>Slide 2 content</div>
    <div data-slider-item>Slide 3 content</div>
  </div>
  <button data-slider-prev>&#8249;</button>
  <button data-slider-next>&#8250;</button>
  <div data-slider-dots></div>
</div>

Listening to Slide Events

Hook into slider events for custom behaviour:

const slider = document.querySelector('[data-slider]');

slider.addEventListener('battersea:slideChange', (e) => {
  console.log('Current slide:', e.detail.currentIndex);
  console.log('Previous slide:', e.detail.previousIndex);
  console.log('Total slides:', e.detail.totalSlides);
});
