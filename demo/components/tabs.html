HTML is the backbone of every web page. It defines the structure and content — headings, paragraphs, links, images, and forms. Everything you see on a page starts here.

It's declarative rather than procedural: you describe what should be on the page, not how to put it there.

CSS handles the visual side of things — colours, spacing, typography, layout, and animations. It takes the raw HTML structure and makes it look the way you want.

Modern CSS is remarkably powerful. Flexbox, Grid, custom properties, and container queries mean you can build complex, responsive layouts without a single line of JavaScript.

JavaScript adds interactivity. It's what makes buttons respond to clicks, forms validate input, and content update without a page reload.

This tabs component is written in vanilla JavaScript — no frameworks, no dependencies. Just the browser APIs you already have.

Installation

Include the required files in your HTML:

<link rel="stylesheet" href="battersea-library.css">
<script src="battersea-utils.js"></script>
<script src="battersea-core.js"></script>
<script src="battersea-tabs.js"></script>

Add the tab markup to your page and it initialises automatically. No JavaScript configuration needed.

Data Attributes

The component uses data attributes for all its configuration:

  • data-tabs — The container element
  • data-tabs-nav — Wraps the tab buttons
  • data-tab="id" — Each tab button, with a unique ID
  • data-tabs-content — Wraps the content panels
  • data-tab-content="id" — Each panel, matching its tab's ID

The IDs link each button to its panel. They can be any string — just make sure they match.

Custom Events

The component dispatches a battersea:tabChange event whenever a tab is switched:

const tabs = document.querySelector('[data-tabs]');

tabs.addEventListener('battersea:tabChange', (e) => {
  console.log('Switched to:', e.detail.tabId);
});

This is useful for loading content on demand, updating URLs, or syncing with other components.

Customising the Look

The default styling is intentionally minimal — a bottom border on the active tab and not much else. Override the CSS to match your design:

[data-tabs-nav] [data-tab] {
  padding: 0.75rem 1.5rem;
  font-size: 0.95rem;
}

[data-tabs-nav] [data-tab].active {
  border-bottom-color: #667eea;
  color: #667eea;
}

Configuration & Options

Data Attributes

Configure the tabs component using these data attributes:

Attribute Element Description
data-tabs Container Initialises the tabs component on the container element
data-tabs-nav Nav wrapper Wraps the tab buttons. Must be a direct child of the container
data-tab="id" Button A tab button. The value is the unique ID that links it to its content panel
data-tabs-content Content wrapper Wraps the content panels. Must be a direct child of the container
data-tab-content="id" Panel A content panel. The value must match a data-tab button ID

Required Structure

The component expects this HTML structure:

<div data-tabs>
  <div data-tabs-nav>
    <button data-tab="tab1" class="active">Tab 1</button>
    <button data-tab="tab2">Tab 2</button>
    <button data-tab="tab3">Tab 3</button>
  </div>
  <div data-tabs-content>
    <div data-tab-content="tab1" class="active">Content 1</div>
    <div data-tab-content="tab2">Content 2</div>
    <div data-tab-content="tab3">Content 3</div>
  </div>
</div>

CSS Classes

The component manages these classes automatically:

Class Element Description
active data-tab button Applied to the currently selected tab. Add manually to pre-select a tab.
active data-tab-content panel Applied to the visible content panel. Add manually to pre-show a panel.

Usage Examples

Basic Tabs

The simplest tab setup:

<div data-tabs>
  <div data-tabs-nav>
    <button data-tab="one" class="active">First</button>
    <button data-tab="two">Second</button>
  </div>
  <div data-tabs-content>
    <div data-tab-content="one" class="active">
      First panel content
    </div>
    <div data-tab-content="two">
      Second panel content
    </div>
  </div>
</div>
Pre-selected Tab

Add active to both the button and its content panel to pre-select a tab:

<div data-tabs>
  <div data-tabs-nav>
    <button data-tab="info">Info</button>
    <button data-tab="details" class="active">Details</button>
  </div>
  <div data-tabs-content>
    <div data-tab-content="info">Info content</div>
    <div data-tab-content="details" class="active">
      This panel is visible on load
    </div>
  </div>
</div>
Listening to Tab Changes

Use the custom event to respond when a user switches tabs:

const tabs = document.querySelector('[data-tabs]');

tabs.addEventListener('battersea:tabChange', (e) => {
  const activeTab = e.detail.tabId;
  console.log('Now viewing:', activeTab);

  // Load content on demand, update the URL,
  // or sync with another component
});
Custom Styling

Override the default styles to match your design. The component uses data attribute selectors, so specificity is low and easy to override:

/* Pill-style tabs */
[data-tabs-nav] {
  border-bottom: none;
  gap: 0.5rem;
}

[data-tabs-nav] [data-tab] {
  border-radius: 20px;
  padding: 0.5rem 1.25rem;
  border: 1px solid #ddd;
}

[data-tabs-nav] [data-tab].active {
  background: #667eea;
  color: white;
  border-color: #667eea;
}
