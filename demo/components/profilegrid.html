Key Features

[]

Responsive Grid

Four columns on desktop, three on tablet, two on mobile. The column count is configurable with a single data attribute.

~

Hover Overlays

Name and title appear smoothly over each photo on hover. Clean and unobtrusive until you need them.

✎

Detailed Lightbox

Click any profile to open a side-by-side panel with photo, contact details, bio, and social links. Stacks vertically on mobile.

☷

Filterable

Configurable tag filters let users narrow the grid by position, company, university, or any custom field you define.

</>

Rich Bio Content

Bios live in a hidden HTML container, so you can include paragraphs, links, lists, or any markup you need.

✔

Graceful Fallbacks

Missing a phone number or university? The component skips any field that is not provided. No empty rows, no errors.

Configuration

Grid Options

Set these on the main container element:

Attribute Values Default Description
data-profile-grid - required Initialises the Profile Grid component
data-profile-grid-columns Number 4 Number of columns on desktop
data-profile-grid-filter Comma-separated field names none Which fields to generate filter pills for (e.g. position,company)

Profile Item Attributes

Set these on each data-profile-item element. All are optional except data-profile-item itself:

Attribute Description
data-profile-name Person's full name (shown on hover and in lightbox)
data-profile-title Job title (shown on hover and in lightbox)
data-profile-position Role or position level (e.g. Director, Engineer)
data-profile-company Company or organisation name
data-profile-university University or educational institution
data-profile-phone Phone number (rendered as a clickable tel: link)
data-profile-email Email address (rendered as a clickable mailto: link)
data-profile-website Website URL (opens in new tab)
data-profile-social JSON array of social links: [{"platform":"linkedin","url":"..."}]

Bio Content

Place a hidden container inside each profile item for rich bio content:

<div data-profile-item data-profile-name="Jane Smith" ...>
  <img src="photo.jpg" alt="Jane Smith">
  <div data-profile-bio>
    <p>First paragraph with <a href="#">links</a>.</p>
    <p>Second paragraph.</p>
  </div>
</div>

Social Platforms

These platforms get their own icon automatically: linkedin, twitter, instagram, facebook, github. Anything else shows a generic link icon with the platform name as a label.

Usage Examples

Basic Grid (No Filters)

The simplest setup. Just a grid of profiles with hover overlays and lightbox:

<div data-profile-grid data-profile-grid-columns="3">
  <div data-profile-item
       data-profile-name="Jane Smith"
       data-profile-title="Designer"
       data-profile-email="jane@example.com">
    <img src="photo.jpg" alt="Jane Smith">
  </div>
  <!-- more items -->
</div>
With Filters

Add the filter attribute to generate clickable pills:

<div data-profile-grid
     data-profile-grid-columns="4"
     data-profile-grid-filter="company,position">
  <div data-profile-item
       data-profile-name="Jane Smith"
       data-profile-company="Acme Corp"
       data-profile-position="Director">
    <img src="photo.jpg" alt="Jane Smith">
  </div>
  <!-- more items -->
</div>

The component scans all items, collects unique values for each filter field, and renders pills automatically.

Rich Bio Content

The bio container is hidden in the grid but cloned into the lightbox when opened. Use any HTML you like:

<div data-profile-item data-profile-name="Jane Smith">
  <img src="photo.jpg" alt="Jane Smith">
  <div data-profile-bio>
    <p>First paragraph of bio text.</p>
    <p>Second paragraph with a
       <a href="https://example.com">link</a>.</p>
    <ul>
      <li>Bullet points work too</li>
    </ul>
  </div>
</div>
Social Links

Pass social links as a JSON array in the data attribute:

<div data-profile-item
     data-profile-name="Jane Smith"
     data-profile-social='[
       {"platform":"linkedin","url":"https://linkedin.com/in/jane"},
       {"platform":"github","url":"https://github.com/jane"},
       {"platform":"portfolio","url":"https://jane.design"}
     ]'>
  <img src="photo.jpg" alt="Jane Smith">
</div>

Known platforms (linkedin, twitter, instagram, facebook, github) get SVG icons. Unknown ones get a link icon with the platform name as text.

Listening to Events

The component fires custom events you can listen for:

const grid = document.querySelector('[data-profile-grid]');

grid.addEventListener('battersea:profileOpen', (e) => {
  console.log('Opened profile:', e.detail.name);
});

grid.addEventListener('battersea:profileClose', () => {
  console.log('Profile lightbox closed');
});
