Key Features

1..n

Smart Page Range

Shows all pages when there are 7 or fewer. For larger sets, it displays the current page with neighbours and ellipsis gaps.

#

Jump to Page

Optional number input lets users type a page number and jump straight to it. Handy for large data sets.

{ }

Server-Side Mode

Fires a custom event on page change so you can fetch data from any backend — PHP, .NET, Node, or a REST API.

A11y

Accessible

ARIA labels, aria-current on the active page, a live region for screen reader announcements, and full keyboard support.

10|25

Page Size Selector

Optional dropdown lets users choose how many items to show per page. Resets to page 1 when changed.

➤

Auto-Initialisation

Add data attributes to your HTML and pagination works immediately. No JavaScript configuration required for client-side mode.

Configuration & Options

Data Attributes

Configure the pagination component using these data attributes:

Attribute Values Default Description
data-pagination — required Activates the pagination component
data-pagination-target CSS selector — Content container whose children are paginated (enables client-side mode)
data-pagination-total-items number 0 Total item count (required for server-side mode; auto-calculated in client mode)
data-pagination-page-size number 10 Items shown per page
data-pagination-mode client / server auto Explicit mode override. Auto-detects based on whether a target container exists
data-pagination-show-info true / false true Show "Showing X–Y of Z" info text
data-pagination-show-input true / false false Show the go-to-page number input
data-pagination-show-first-last true / false false Show first and last page buttons
data-pagination-show-sizes comma-separated numbers — Page size options for a dropdown selector, e.g. "5,10,25,50"
data-pagination-scroll-top true / false false Scroll the target container into view when the page changes

Custom Events

The component dispatches this event on every page change:

Event Detail Description
battersea:pagination:change { page, pageSize, totalPages, totalItems } Fires on the host element when the current page changes. Listen to this for server-side data fetching.

Public Methods

Access the component instance via Battersea.getInstance(el):

Method Parameters Description
setTotalItems(total) number Update the total item count (useful after an AJAX response)
setPage(page) number Jump to a specific page programmatically
refresh() — Re-collect items and re-render (call after adding/removing content in client mode)

Usage Examples

Client-Side Pagination
<!-- Content container -->
<div id="my-cards">
  <div>Card 1</div>
  <div>Card 2</div>
  <!-- ... more items -->
</div>

<!-- Pagination control -->
<div data-pagination
     data-pagination-target="#my-cards"
     data-pagination-page-size="6"
     data-pagination-show-info="true">
</div>
Server-Side with AJAX
<div id="results"></div>

<div id="my-pagination"
     data-pagination
     data-pagination-mode="server"
     data-pagination-total-items="250"
     data-pagination-page-size="20"
     data-pagination-show-info="true"
     data-pagination-show-input="true">
</div>

<script>
var paginationEl = document.getElementById('my-pagination');

paginationEl.addEventListener('battersea:pagination:change', function(e) {
  var page = e.detail.page;
  var size = e.detail.pageSize;

  // Fetch from your API
  fetch('/api/items?page=' + page + '&size=' + size)
    .then(function(res) { return res.json(); })
    .then(function(data) {
      // Render your results
      document.getElementById('results').innerHTML = data.html;

      // Update total if it changed
      var instance = Battersea.getInstance(paginationEl);
      if (instance) instance.setTotalItems(data.total);
    });
});
</script>
