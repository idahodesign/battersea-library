ProgressBar Component

Horizontal and circular progress bars with scroll-triggered animation and percentage display

Live Demonstrations

Scroll down to trigger the animations

Horizontal Progress Bars

Classic horizontal bars at different values. They animate when they scroll into view, so if you don't see them moving, scroll down a bit and they'll kick in.

Horizontal with Labels

Add a label to each bar so people know what they're looking at. Great for showing skill levels, project progress, or anything else that benefits from a name.

Circular Progress Bars

Same idea, different shape. These circular bars work well for dashboards, stats sections, or anywhere you want something a bit more visual.

Circular with Different Stroke Widths

Control the thickness of the circular bars with the stroke attribute. Thinner strokes look more elegant, thicker ones are bolder and easier to read at a glance.

Key Features

—

Horizontal Bars

Classic horizontal progress bars that fill from left to right. Simple, readable, and familiar.

◯

Circular Bars

SVG-based circular progress indicators. Configurable size and stroke width for the right look.

~

Scroll-Triggered

Animations trigger when the bars scroll into the viewport. Users see the progress fill up in real time.

%

Percentage Display

Optionally show the current percentage value inside or beside the bar. Numbers count up as the bar fills.

Aa

Custom Labels

Add text labels to identify what each bar represents — skills, project phases, loading states, whatever you need.

➤

Zero Config

Everything is controlled through data attributes. Drop in the markup and it just works — no JavaScript setup required.

Configuration & Options

Data Attributes

Configure each progress bar using these data attributes:

Attribute Values Default Description
data-progress - required Initialises the progress bar component on this element
data-progress-type horizontal / circular horizontal The style of progress bar to render
data-progress-value 0–100 0 The percentage value the bar should fill to
data-progress-label String - A text label displayed alongside the progress bar
data-progress-show-percent true / false false Show the percentage number as the bar animates
data-progress-animated true / false true Whether the bar animates on scroll or starts filled
data-progress-size Number (px) 120 Diameter of circular progress bars (circular only)
data-progress-stroke Number (px) 8 Stroke width of circular progress bars (circular only)

Markup Structure

The markup is dead simple — just a single div with data attributes. The component builds everything else internally:

<!-- Horizontal -->
<div data-progress
     data-progress-type="horizontal"
     data-progress-value="75"
     data-progress-label="CSS"
     data-progress-show-percent="true"
     data-progress-animated="true">
</div>

<!-- Circular -->
<div data-progress
     data-progress-type="circular"
     data-progress-value="80"
     data-progress-size="140"
     data-progress-stroke="10"
     data-progress-show-percent="true"
     data-progress-animated="true">
</div>

Static (No Animation)

If you don't want the scroll-triggered animation, set data-progress-animated="false" and the bars will appear already filled:

<div data-progress
     data-progress-type="horizontal"
     data-progress-value="80"
     data-progress-label="Complete"
     data-progress-show-percent="true"
     data-progress-animated="false">
</div>
Custom Styling

Override the default colours and appearance with your own CSS:

/* Change the bar colour */
[data-progress] .battersea-progress__fill {
  background: linear-gradient(90deg, #667eea, #764ba2);
}

/* Change the track background */
[data-progress] .battersea-progress__track {
  background: #f0f0f0;
  border-radius: 8px;
}

/* Style the percentage text */
[data-progress] .battersea-progress__percent {
  font-weight: 700;
  colour: #333;
  color: #333;
}

/* Circular bar colour */
[data-progress] .battersea-progress__circle-fill {
  stroke: #4ecdc4;
}
