Page Navigation Component

Prev/next page links that read your site's menu and work out where to go

Live Demonstrations

Each example below uses a different navigation mode

All Pages (Sequential)

Moves through every page in the site, one by one. The category name appears above the page title so you can see which section you're heading into.

Within Category

Stays within the current category. If you're in the Navigation section, it only moves between Navigation pages. Handy when you want people to explore related content.

Between Categories

Jumps between top-level sections. Useful when you want to give people a way to move across major areas of the site.

Arrows Only

A stripped-back version with no page titles. Just the arrows. On mobile, all modes look like this automatically.

Configuration & Options

Data Attributes

Configure the component using these data attributes:

Attribute Values Default Description
data-pagenav - required Initialises the page navigation component on the element
data-pagenav-mode pages, category, categories pages Which navigation sequence to use
data-pagenav-show-title true, false true Whether to display the page name alongside the arrow
data-pagenav-show-category true, false false Show the parent category name above the page title
data-pagenav-label Any string none Optional label text displayed between the prev/next links

CSS Custom Properties

Override these variables to customise the appearance:

Variable Default Description
--battersea-pagenav-link #667eea Link colour
--battersea-pagenav-link-hover #4c5fd5 Link hover colour
--battersea-pagenav-arrow-size 1.5rem Size of the arrow characters
--battersea-pagenav-title-size 0.95rem Font size of the page title
--battersea-pagenav-title-weight 500 Font weight of the page title
--battersea-pagenav-category-size 0.75rem Font size of the category label
--battersea-pagenav-category-color #999 Colour of the category label
--battersea-pagenav-padding 1.5rem 0 Padding around the component
--battersea-pagenav-border 1px solid #e0e0e0 Top border of the component
--battersea-pagenav-margin 2rem Top margin of the component
--battersea-pagenav-label-color #999 Colour of the centre label text

Dependencies

The PageNav component requires the NavData service, which reads the site's navigation menu. Make sure battersea-nav-data.js is loaded before battersea-pagenav.js.

Usage Examples

Basic Page Navigation

Drop this on a page and it generates prev/next links through all pages in order:

<div data-pagenav></div>
Within Category Navigation

Keep navigation within the current section only:

<div data-pagenav data-pagenav-mode="category"></div>
With Category Label and Centre Text

Show the category name above the page title and add a label between the links:

<div data-pagenav
  data-pagenav-mode="pages"
  data-pagenav-show-category="true"
  data-pagenav-label="Browse pages"></div>
Arrows Only

Hide the page titles for a minimal look:

<div data-pagenav data-pagenav-show-title="false"></div>
Required Scripts

Include the core files, NavData service, and the page navigation script:

<script src="battersea-utils.js"></script>
<script src="battersea-core.js"></script>
<script src="battersea-nav-data.js"></script>
<script src="battersea-pagenav.js"></script>

The NavData service must load before the PageNav component. It parses the navigation menu and makes the data available for the prev/next links.

Accessibility Features

Built with inclusivity in mind

nav

Landmark Navigation

Uses <nav> with aria-label="Page navigation" so screen readers can identify it.

➤

Descriptive Labels

Each link has an aria-label like "Previous: Header" or "Next: Horizontal Navigation" so users know exactly where they'll go.

Tab

Keyboard Navigable

Standard anchor links -- Tab to reach them, Enter to follow. Nothing unusual to learn.

[ ]

Focus Indicators

Visible focus outlines on all links for keyboard navigation.
